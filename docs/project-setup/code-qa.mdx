# Eslint

Eslint is the fastest way to spot errors on our code as we write.

[Docs](https://eslint.org/docs/latest/use/)

Example: [CCSA eslint configuration](https://github.com/Vizzuality/ccsa/blob/main/client/.eslintrc.json)

Note: do not forget to tell Eslint what files/folders should be ignored, otherwise it might freeze trying to analyze codes from other parts like node_modules where you can find a lot of minified code.

[Docs for eslintignore](https://eslint.org/docs/latest/use/configure/ignore)

Example: [CCSA eslintignore](https://github.com/Vizzuality/ccsa/blob/main/client/.eslintignore)

For a better DX, the [VS Code extension](https://github.com/Vizzuality/ccsa/blob/main/client/.eslintignore) is strongly recommended.
